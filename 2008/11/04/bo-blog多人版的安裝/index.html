Bo-Blog多人版的安裝

1.下載軟體：#wget http://xoops.tnc.edu.tw/uploads/bo\_blog.zip

2.#unzip bo_blog.zip將檔案放到xoops資料夾下，我是放在 /var/www/html
/var/www/html/blog <—-原始程式
/var/www/html/modules/bo_blog <—-模組程式

3.開啟blog資料夾的寫入權限：chmod 777 blog <—-原始程式

4.更改模組的語系為utf8（若系統使用big5，則省略此步驟。）
#cd /var/www/html/modules/bo_blog/language
#mv tchinese tchinese_big5
#mv tchinese_utf8 tchinese

5.用系統管理者身份登入XOOPS後台，安裝多人版bo-blog模組

6.安裝完成後，會回到bo-blog模組畫面，請記得要先去後台設定mysql的管理者帳號和密碼。偏好設定/多人Bo-Blog/模組偏好設定。這裡需要設定管理MySQL資料庫的「帳號」和「密碼」

7.到模組/多人Bo-Blog/主管理介面，然後按「回模組首頁」，應該會看到「啟動我的部落格」按鈕，按一下該按鈕啟動部落格。

這樣就安裝好了嗎？好戲才正要上演呢！在按下啟動部落格的瞬間，畫面傳來下面的錯誤訊息，連原本安裝好的XOOPS都無法瀏覽了！
This page cannot be displayed due to an internal error.
If you are the administrator of this site, please visit the Xoops Troubleshooting Page for assistance.
Helpful infomation you should provide while asking for assistance:
Error [Xoops]: Unable to connect to database in file class/database/databasefactory.php line 34

8.重新建立安裝XOOPS時的帳戶：
原來，在啟動Bo-Blog的同時，安裝XOOPS時的MySQL帳戶的密碼被改掉了。此時，用phpMyAdmin登入資料庫管理畫面，進入『權限』項目，將原來安裝XOOPS時的帳戶給刪除，再建立一個一樣的帳戶，（帳戶和密碼必須要和原來安裝XOOPS時的帳戶一樣。）建好登出後，XOOPS恢復了瀏覽的畫面了！

接著，進入xoops後台，多人Bo-Blog主管理介面，點選部落格，出現了另一串錯誤訊息。真是好事多磨也…
Bo-Blog Database System Tips: Can not connect to MySQL server
Time: 2008-5-31 11:12am
Script:
Error: Access denied for user ‘root‘@’localhost’ (using password: YES)
Errno.: 1045

9.更改個人部落格內的設定檔：
#nano /var/www/html/blog/1/data/config.php
第4行有紀錄mysql密碼，我看到一長串密碼，但不是我設定的MySQL帳戶的密碼，所以我把他改成和xoops中設定的密碼。
$db_password=”XXXXXXX”;
這樣就可以囉！

10.修正驗證碼無法顯示的問題：（系統必須安裝GD函數資料庫）
var/www/html/xoops/blog/source/inc/securitycode.php
把require_once (“../data/config.php);
改成require_once( “/var/www/html/blog/1/data/config.php”);
存檔後即可使用驗證碼。

11.修正進入相簿後一片空白的問題：
#cd /var/www/html/blog/source/plugin
#chmod 777 DDRR_Album/
#chmod 777 DDRR_Album/template_c/
#chmod 777 DDRR_Album/config/
#chmod 777 DDRR_Album/cache/
#chmod 666 DDRR_Album/config/config_xml.php

※以上資料摘錄自『學鋼琴的老師』：http://media.tpes.hcc.edu.tw/xoops/blog/1
